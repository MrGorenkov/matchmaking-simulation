Дискретно-событийная имитационная модель системы подбора матчей многопользовательской игры. Курсовая работа по дисциплине «Имитационное моделирование систем массового обслуживания».

## Что моделируется

- Поток приходящих в очередь игроков по Пуассоновскому процессу
- Три региона: EU, NA, AS — каждый со своим пулом серверов
- MMR-разброс (skill rating) при подборе, ограничение `mmr_spread_max`
- Региональные очереди с приоритизацией по времени ожидания
- Серверный пул с фиксированной пропускной способностью
- Балансировка нагрузки и переполнение очередей (отказы)

## Сценарии

| Сценарий | λ | Серверов / регион | Размер матча | Сценарий нагрузки |
|---|---|---|---|---|
| Low Load | 0.5 | 2 | 10 | Низкая нагрузка |
| Balanced | 1.0 | 3 | 10 | Сбалансированная |
| High Load | 2.0 | 5 | 10 | Высокая нагрузка |
| Overload | 3.5 | 4 | 10 | Перегрузка |

## Метрики

- Распределение времени ожидания (среднее, медиана, перцентили)
- Загрузка серверов по регионам
- Вероятность отказа из-за переполнения очереди
- Средний MMR-разброс в собранных матчах
- Пропускная способность системы

## Стек

- **Backend** — Flask + Flask-CORS
- **Расчёты** — Python 3, NumPy
- **Визуализация** — Chart.js (распределения, столбчатые диаграммы, тепловые карты)
- **Экспорт** — CSV для дальнейшего анализа

## Запуск

```bash
pip install -r requirements.txt
python app.py
```

Открыть в браузере: <http://127.0.0.1:5000>

## Параметры эксперимента

Все параметры конфигурируются через UI или передаются в API:

- `lambda_rate` — интенсивность Пуассоновского потока (игроков/сек)
- `sim_time` — длительность симуляции (сек)
- `max_queue` — максимальный размер очереди до отказа
- `max_wait_time` — максимальное время ожидания
- `servers_eu / servers_na / servers_as` — пул серверов по регионам
- `match_size` — число игроков в матче
- `service_time` — длительность матча
- `mmr_spread_max` — максимальный допустимый разброс MMR
- `runs` — число повторов эксперимента для усреднения
